    Sample 1 : multiple upload
    Upload one or more files to the server.

    In this sample, the files are saved directly with their original name to a virtual directory on the web server.

    Sample 2 : multiple upload using the «files» collection
    Upload one or more files to the server and recover and display details of files sent.
    The html form allows the user to select the file(s). The JSP file uses jspSmartUpload to save the files to the server’s disk and to display the details of each file (name, extension, size, type, etc.).

    In this sample, we examine the «Files» collection. For each file on the form which is actually sent, the corresponding properties are displayed and the file is saved to a virtual directory on the Web server.

    Sample 4 : multiple upload to a database
    Upload one or more files to a database on the server. The html form allows the user to select the file(s). The JSP file uses jspSmartUpload to save the files in a database (Access in this sample).

    This sample use a mySQL Database define by this scripts :

    CREATE TABLE TFILES (ID INT(4) NOT NULL, FILENAME VARCHAR(255), FILE LONGBLOB, PRIMARY KEY (ID));

    INSERT TABLE TFILES (ID,FILENAME,FILE) VALUES (1,"test",null);


    The files are saved in the «FILE» field of the «TFILES» table and a new recording is generated for each file sent.
