Web Server--Key Technology as New Kind of App Server
By John Hoagland, Components Online, (draft).

The big technological story is the integration of client/server by the web. What was once a difficult technology to implement has become ubiquitous. One form of client/server, the application server, has "reinvented" itself on the web.

The internet's TCP/IP has taken over as the network protocol of choice, even as LAN's have been subsumed by intranets. Basically, the NOS, or Network Operating System, has replaced the OS, since any computer bought today is equipped with internet technologies.

As a result, any contemporary computer can function as a client or server depending on the software it runs, at least on the low end of the server market, such as an intranet. In many instances, the network "link" is the constraining bandwidth/cost factor rather than the computer itself.

Shift to the Server Side

After all the attention to "thin" and other clients, there is a shift to the server side in the pursuit of the next stage of the quest for interoperability, the lodestar of computing.

The reigning players in server software, with the hope of parlaying the browser example into the application sphere in order to replace the PC industry's dominance of the client world, have focused on the platform-independent appeal of Java and overlooked the importance of their own server backyard, i.e. the implications of the web for server software.

Important categories of server software is being re-written to function in a web environment as e-commerce emerges to drive the market. Again, Java is being positioned as the magic portion, this time aligned with CORBA and EJB middleware, rather than the browser, to ward off assaults on the server markets by what was the PC industry, itself being transformed in the process.

The key fear on all fronts is which product will be "commoditized", i.e. forced to compete on price alone? Will it be the OS? Will it be the server "box"--made by what the PC world calls the OEM? Will it be software products in the same way as PC hardware products?

The battle for these server markets has been joined and the key technology is the WEB SERVER.

The web server, strictly speaking the software for presenting HTTP-based "screens" to the viewer, is becoming the key technology as the "middle tier" where the business logic is executed for all kinds of services to web clients via the browser.

The web server appears to be the focal point due to the need for practical interoperability within the enterprise. Since the desktop/workstation has been integrated on the network by the browser via the web server, connectivity to database servers are being linked by the web server as well. Thus all roads, at least the important ones, lead through the web server.

Web server "farms" are the means by which different server functionality is distributed among related servers in a group of servers, collectively called a "farm," or even more tightly integrated in clusters.

Clustering is the hardware-based means of achieving scalability. Scaling "up" has been accomplished through CPUs clustered via SMP (symmetrical processing), involving the tightest physical proxmity, while scaling "out" may have more widely distributed "nodes," each of which could be a SMP cluster.

Server software can be distinguished between the software that is used to administer the server itself and that which is used to manage the business running on the server. An example of the former are Online Transaction Processing (OLTP) applications that do the back end work of a TP monitor as well as a programming infrastructure, such as an ORB (Object Request Broker) for a componentized system (see Ecommerce Transactions). An example of the latter would be ERP. App Server terminology is currently broad enough to include both, adding to the confusion in this area.

Application Servers Gain New Meaning

Once a description of departmental servers that supplied the desktop with traditional apps over LANs, application servers have now gained an entirely new meaning in the context of the web. Now the term describes the services supplied to enable new "applications" using web technologies, as well as the traditional apps offered on a "service plan."

While this development represents the integration of traditional client/server with the new web client/server, most of the action is in supplying these services via web servers.

Lost in the brouhaha over NCs v. PCs, and the assumption that the servers in general would get greater business, with the web server being merely the generic cousin of the application and database servers.

In short, the technology has moved beyond the OS, and the middleware-based NOS is upon us and rapidly evolving in the market.
